## Railway Crash Incident (02:38–02:50 AM)

### Root Cause
Pushing to main triggered Railway redeploy of ALL services in the monorepo. 4 AdonisJS services (auth, order, cash, reservaciones) crashed with `Cannot find module '/app/bin/server.js'`.

Railway uses **Railpack** (not Dockerfile) as builder. Railpack runs `npm start` directly, ignoring the Dockerfile's `CMD`. The `package.json` had `"start": "node bin/server.js"` (TS source) but compiled JS lives at `build/bin/server.js`.

### Fix
Commit `819def23`: Changed `"start": "node build/bin/server.js"` in all 4 package.json files. All 4 services back online with HTTP 200.

### Lesson Learned
- Railway's Railpack builder ignores Dockerfile CMD — always ensure `package.json` start script matches the compiled output path
- Monorepo pushes redeploy ALL services — need watch paths configured per service to prevent unnecessary redeployments
- The previous deploys were "Removed" status — Railway was running old container images. The new push forced a fresh build that exposed the existing bug.
