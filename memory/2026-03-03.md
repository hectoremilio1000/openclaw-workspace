## Inventory Presentations UX Overhaul (afternoon session)

### Problem
Chef and Jefe de Barra confused by presentations system:
- Items like "Perrier 380ml" with base unit ml → system created "1 ml" presentation (useless)
- They buy in boxes (caja de 24) but needed individual piece cost for recipes
- In recipes, they want to say "300 ml" and know the cost, not calculate 1/12 of a box

### Solution — 6 commits pushed to pos-front main
1. **Tags in presentation drawer** (`ItemPresentationsDrawer.tsx`): 🔹Receta (cyan) for unit presentations, 📦Compra (orange) for purchase presentations
2. **Recipe auto-select** (`RecipeEditorDrawer.tsx`): Auto-selects unit presentation (content=1 or smallest) instead of purchase. Dropdown shows unitarias first with 🔹/📦 emojis
3. **Recipe unit converter** (`RecipeEditorDrawer.tsx`): Live conversion showing "300 ml = 0.79 porciones", cost estimate, cost per base unit
4. **Purchase calculator** (`GeneralTab.tsx`): When "Presentación de compra" is toggled, shows pieces × content per piece = total calculator
5. **Auto-create unit from purchase** (`PresentationEditModal.tsx`): When saving a purchase presentation with pieces/content info, auto-creates the unit presentation with derived cost (box_cost / pieces)
6. **Piece content field on insumo** (`InventoryItemFormDrawer.tsx`): For volume/weight base units (ml, l, g, kg), shows "¿Cuánto contiene cada pieza?" field. Works on both create AND edit.

### Final Flow (after iterations with user)
1. Create insumo "Perrier 380ml", base unit = ml, piece content = 380 → NO auto presentation
2. Open presentations → Create "Caja de 24" via purchase calculator (24 × 380 = 4,560 ml, cost $490)
3. System auto-creates "1 pieza (380 ml)" with cost = $490/24 = $40.83
4. In recipe: select Perrier → auto-selects "1 pieza (380ml)" → put 300 ml → shows "= 0.79 porciones, costo $32.24"
