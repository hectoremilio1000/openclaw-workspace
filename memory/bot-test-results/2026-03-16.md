## 🔴 ALERT: L-legal category score 27.5 (< 70)

### Diagnosis

The bot has **zero RAG content for legal/regulatory questions** about Mexican food safety standards (NOM-251, Distintivo H, etc.). Two failure modes:

1. **L01 (NOM-251)**: Intent correctly classified as "Pregunta de negocio o legal" but no knowledge base content exists → generic "no context" fallback
2. **L11 (Distintivo H)**: Intent **misclassified** as "informacional" (system navigation) instead of legal/regulatory → completely wrong response asking about which module the user is in

### Root Causes

1. **Missing RAG documents**: The bot's knowledge base doesn't include Mexican food safety regulations (NOM-251, NOM-093, Distintivo H, COFEPRIS requirements)
2. **Intent classification gap**: "Distintivo H" isn't recognized as a legal/regulatory topic — the classifier falls back to "informacional" (how-to-use-the-system)
3. **Poor fallback UX**: When no RAG content matches, the bot should at minimum say "I don't have that information but here's where to look" instead of a dead-end response

### Recommended Fixes

1. **Add regulatory RAG docs**: Upload NOM-251, NOM-093, Distintivo H guides to the bot's knowledge base
2. **Improve intent classifier**: Add "legal/regulatory" training examples including "Distintivo H", "NOM-251", "COFEPRIS", "salubridad"
3. **Better fallback**: When intent=legal but no RAG match, respond with "No tengo esa información en mi base de datos, pero puedes consultar [COFEPRIS/Secretaría de Salud]" instead of a dead-end
