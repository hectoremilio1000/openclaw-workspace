## 🔴 ALERT: L-Legal Category Below 70 (Score: 46.0)

### Root Cause
The bot has **zero coverage for regulatory/legal questions** (NOM-251, Distintivo H, food safety regulations). Both L01 and L11 triggered the generic "module navigation" fallback instead of an honest "I don't have that information" response.

### Issues
1. **Missing RAG content:** No regulatory/legal documents in the knowledge base
2. **Wrong fallback template:** When informational intent has no RAG match, it says "Dime en que modulo estas" — which is confusing and misleading for non-system questions
3. **No graceful degradation:** Bot should say "No tengo información sobre ese tema, pero puedo ayudarte con operación del restaurante" instead

### Recommended Fixes
1. **Quick fix:** Change the informational fallback to: "No tengo información sobre ese tema. Puedo ayudarte con reportes, ventas, inventario, turnos y operación del restaurante."
2. **Medium-term:** Add key regulatory docs to RAG (NOM-251, Distintivo H, COFEPRIS basics, etc.)
3. **Nice-to-have:** Classify "legal/regulatory" as a sub-intent and handle it explicitly

### A-Reportes Below 85 (Score: 80.8)
The A11 test ("productos más vendidos hoy") failed to extract "hoy" from the message on the first attempt, asking for a date that was already provided. This is a date parsing inconsistency — works sometimes but not always. Investigate the date extraction pipeline for the "productos más vendidos" intent.
