## 6:20-6:49 PM — Anthropic/Claude Auth Fix

### Problem
- Claude Opus 4.6 failing with: "Third-party apps now draw from your extra usage, not your plan limits"
- Sonnet worked fine; issue was specific to how auth was configured

### Root Cause
- Agent main only had `anthropic:manual` (token type) in auth-profiles.json
- No `anthropic:default` (api_key) existed in the agent's auth
- The manual token was being rejected by Anthropic's new third-party usage policy

### Fix Applied
1. Backed up auth-profiles.json and openclaw.json
2. Removed all old Anthropic auth entries (surgical, kept OpenAI/Codex intact)
3. Ran `openclaw models auth setup-token --provider anthropic`
4. User generated new token via `claude setup-token` and pasted it
5. New `anthropic:default` profile created with fresh token
6. Gateway restarted

### Result
- **Claude Opus 4.6 now working** with new token
- User confirmed: "ya funciona con ese cambio de token"

### Key Lesson
- When Anthropic changes billing/policy for third-party apps, old tokens may stop working
- Fix is: reset auth profile + generate fresh token via `claude setup-token`
- Auth profiles live in `~/.openclaw/agents/main/agent/auth-profiles.json`
